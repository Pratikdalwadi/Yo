    Overview

    DataExtract Pro is a comprehensive web application for intelligent document data extraction using AI/ML technologies. The application processes various document formats (PDF, images, Word documents) and extracts structured data using multiple AI providers including OpenAI GPT-5 and Google Gemini. It features a React frontend with shadcn/ui components, an Express.js backend, and PostgreSQL database with Drizzle ORM for data persistence.

    User Preferences

    Preferred communication style: Simple, everyday language.

    System Architecture

    Frontend Architecture

    • Framework: React 18 with TypeScript and Vite for build tooling
    • UI Library: shadcn/ui components built on Radix UI primitives for accessibility
    • Styling: Tailwind CSS with custom design tokens and dark mode support
    • State Management: TanStack Query (React Query) for server state and API caching
    • Routing: Wouter for lightweight client-side routing
    • Forms: React Hook Form with Zod validation for type-safe form handling

    Backend Architecture

    • Runtime: Node.js with Express.js framework using TypeScript
    • API Design: RESTful API with structured error handling and request logging middleware
    • File Processing: Multer for multipart file uploads with configurable storage and validation
    • Document Processing: Custom DocumentProcessor service supporting PDF parsing, image processing with Sharp, and multi-format text extraction
    • AI Integration: Modular AI provider system with factory pattern supporting OpenAI GPT-5 and Google Gemini APIs

    Database Design

    • ORM: Drizzle ORM with PostgreSQL dialect for type-safe database operations
    • Schema: Four main entities - users, documents, extraction jobs, and schema templates
    • Storage Strategy: Hybrid approach with file metadata in database and actual files on filesystem
    • Migration Management: Drizzle Kit for schema migrations with version control

    Data Processing Pipeline

    • Upload Flow: File validation → storage → metadata persistence → processing queue
    • Extraction Flow: Document parsing → AI provider selection → schema-based extraction → result storage
    • Export System: Multiple format support (JSON, CSV) with customizable options for confidence scores and source mapping

    AI Provider Integration

    • OpenAI Integration: GPT-5 Vision API for multimodal document processing with structured output
    • Google Gemini: Alternative AI provider for document analysis and data extraction
    • Provider Abstraction: Common interface allowing easy addition of new AI services
    • Context Management: Intelligent prompt engineering with schema-aware extraction templates

    Security & File Handling

    • File Validation: MIME type checking and size limits for uploaded documents
    • Storage Security: Isolated upload directory with UUID-based file naming
    • API Security: Request validation with structured error responses
    • Type Safety: End-to-end TypeScript with shared schema definitions

    External Dependencies

    AI Services

    • OpenAI API: GPT-5 Vision model for advanced document understanding and multimodal processing
    • Google Gemini API: Alternative AI provider for document analysis with competitive accuracy

    Database & Storage

    • Neon Database: Serverless PostgreSQL hosting with connection pooling via @neondatabase/serverless
    • Local File Storage: Filesystem-based document storage with configurable upload directory

    Document Processing Libraries

    • PDF Processing: pdf-parse-debugging-disabled for reliable PDF text extraction
    • Image Processing: Sharp for image manipulation, format conversion, and optimization
    • File Upload: Multer middleware for handling multipart form data and file validation

    Development Tools

    • Build System: Vite with React plugin and TypeScript support for fast development builds
    • Code Quality: ESLint and TypeScript for static analysis and type checking
    • Database Tooling: Drizzle Kit for schema management and migration generation
